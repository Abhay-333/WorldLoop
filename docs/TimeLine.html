1. Project Overview

WorldLoop is a full-stack social media platform built on the MERN stack (MongoDB, Express, React, Node.js), with a Vite + TypeScript frontend and a layered Node.js backend. The project was internally developed under the codename "NEXUS" and has since been renamed WorldLoop across product-facing documentation.

Development is organized into two major releases: V1.0, which establishes the core social experience (auth, profiles, posts, feed, engagement, notifications, search), and V2.0, which layers on real-time messaging, ephemeral stories, algorithmic feed ranking, and platform-scale infrastructure.

2. Architecture Summary

2.1 Frontend

  • Feature-Based Architecture (FBA) — each feature is a self-contained vertical slice (components, hooks, api, store, types) exposing a public barrel via index.ts
  • Stack: React 18, Vite, TypeScript, TanStack Query (server state), Redux Toolkit (UI state), Tailwind CSS, shadcn/ui, Socket.io client

2.2 Backend

  • Layered Architecture — strict one-way flow: Route → Controller → Service → Repository
  • Stack: Node.js 20, Express 5, TypeScript, MongoDB/Mongoose, Redis, JWT authentication, BullMQ job queues, Socket.io server

5. Database Collections (Reference)

  • users — profile data, auth provider, privacy settings
  • posts — media, caption, visibility, hashtags/mentions
  • follows — follower/following relationships, request status
  • likes — idempotent like records per post
  • comments — threaded, one reply level
  • notifications — 90-day TTL, BullMQ-driven fanout
  • conversations / messages — introduced in V2.0 for DMs and group chats

6. Assumptions & Notes

  • Timeline assumes one full-stack developer working at a standard pace; compress or extend phases as team size or velocity changes.
  • Phases within each version can run partially in parallel where frontend and backend work are independent (e.g., UI scaffolding while API contracts are finalized).
  • V2.0 phases are sequenced by dependency risk — messaging and stories first (highest user-facing value), infrastructure/scale last.
  • This document should be revisited and updated as actual velocity data becomes available after Phase 1–2.
